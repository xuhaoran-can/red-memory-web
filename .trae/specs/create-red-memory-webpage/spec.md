# 红色记忆AI对话网页 Spec

## Why
创建一个基于济南英雄历史主题的AI对话互动学习平台，通过游戏化元素（积分、勋章、排行榜）提升用户参与度和学习体验。

## What Changes
- 创建完整的前端网页应用，包含AI对话系统、积分系统、勋章收集、排行榜等功能
- 实现响应式布局，适配不同屏幕尺寸
- 集成图表展示和数据可视化

## Impact
- Affected specs: 无（新项目）
- Affected code: 全部文件（HTML/CSS/JavaScript）

## ADDED Requirements

### Requirement: 页面整体布局
系统 SHALL 提供一个三栏式布局的网页界面：
- 顶部导航栏：显示标题、日期、积分和分享功能
- 左侧边栏：荣誉勋章馆和解密活跃度趋势图
- 中间主区域：AI对话交互界面
- 右侧边栏：智能线索墙和排行榜

#### Scenario: 页面加载完成
- **WHEN** 用户打开网页
- **THEN** 显示完整的页面布局，包含所有功能模块
- **AND** 当前日期自动更新显示

### Requirement: 顶部导航栏
系统 SHALL 在页面顶部显示导航信息：
- 标题："★ 红色记忆——济南英雄风云录"
- 当前日期（格式：YYYY年MM月DD日）
- 积分显示（初始值为0）
- 分享成就按钮

#### Scenario: 查看积分
- **WHEN** 用户查看顶部导航栏
- **THEN** 显示当前用户积分数值
- **AND** 积分变化时实时更新

### Requirement: 荣誉勋章馆（左侧）
系统 SHALL 在左侧展示勋章收集系统：
- 显示6个勋章图标（英雄先锋、智勇双全、解密达人等）
- 勋章状态区分（已获得/未获得）
- 当前勋章进度条显示
- 已收集勋章数量统计

#### Scenario: 查看勋章状态
- **WHEN** 用户查看荣誉勋章馆
- **THEN** 显示所有可获得的勋章及其状态
- **AND** 高亮已获得的勋章
- **AND** 显示当前收集进度

### Requirement: AI对话系统（中间主区域）
系统 SHALL 提供核心的AI对话交互功能：
- 红色导览员头像和名称显示
- 对话消息展示区域（支持多轮对话）
- 欢迎消息和引导提示
- "开始解密挑战"按钮触发对话
- 底部输入框用于用户输入消息
- 发送按钮提交用户消息
- 消息气泡样式区分（系统消息/用户消息）

#### Scenario: 开始对话
- **WHEN** 用户点击"开始解密挑战"按钮
- **THEN** AI导览员发送欢迎引导消息
- **AND** 输入框变为可用状态

#### Scenario: 发送消息
- **WHEN** 用户在输入框输入内容并点击发送
- **THEN** 用户消息显示在对话区域
- **AND** AI生成回复并显示
- **AND** 对话区域自动滚动到最新消息

#### Scenario: 双击编辑组件
- **WHEN** 用户双击对话区域的编辑组件
- **THEN** 触发编辑模式或显示相关功能提示

### Requirement: 智能线索墙（右侧）
系统 SHALL 在右侧提供线索查询功能：
- 搜索输入框用于关键词搜索
- "申请调查线索"按钮（消耗10积分）
- 线索结果展示区域

#### Scenario: 申请调查线索
- **WHEN** 用户点击"申请调查线索"按钮且积分≥10
- **THEN** 扣除10积分
- **AND** 显示相关历史线索信息
- **AND** 积分实时更新

#### Scenario: 积分不足申请线索
- **WHEN** 用户点击"申请调查线索"按钮但积分<10
- **THEN** 提示积分不足，无法申请

### Requirement: 今日解密排行榜（右下角）
系统 SHALL 展示实时排名系统：
- 显示前5名用户的排名列表
- 包含用户名和解密数量
- 排名数字高亮显示
- 实时更新排名数据

#### Scenario: 查看排行榜
- **WHEN** 用户查看排行榜区域
- **THEN** 显示当前前5名用户
- **AND** 每行显示排名、用户名、解密数量

### Requirement: 解密活跃度趋势图（左下角）
系统 SHALL 使用图表展示数据统计：
- 折线图显示近7天的解密活跃度趋势
- X轴显示日期，Y轴显示活跃度数值
- 数据点可交互显示具体数值
- 图表自适应容器大小

#### Scenario: 查看趋势图
- **WHEN** 用户查看左下角图表区域
- **THEN** 显示折线图展示活跃度数据
- **AND** 鼠标悬停显示详细数据

### Requirement: 视觉设计规范
系统 SHALL 遵循以下视觉设计要求：
- 主色调：深红色背景（#8B0000系列）
- 强调色：金黄色（#FFD700系列）
- 边框和装饰使用金色描边效果
- 文字颜色：白色/金色搭配
- 卡片式布局带圆角和阴影
- 按钮采用渐变效果和hover动画
- 整体风格庄重典雅，符合红色文化主题

## MODIFIED Requirements
无（新项目）

## REMOVED Requirements
无（新项目）

## Technical Implementation Notes
- 使用纯前端技术实现（HTML + CSS + JavaScript）
- 可选使用 Chart.js 实现图表功能
- 对话系统可使用模拟数据或集成AI API
- 响应式设计确保在不同设备上的良好体验
- 所有交互需有视觉反馈（loading状态、hover效果等）
